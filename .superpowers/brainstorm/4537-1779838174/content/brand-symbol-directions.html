CSC CLI/Ink brand symbol directions

These are not full UI themes. They compare terminal-native memory marks that can be applied to startup, waiting, permission, tool calls, plan progress, and completion states.

A. Railmark
csc  restore session
|   goal      fix failing typecheck
|   action    reading provider registry
|   tool      GrepTool matched 4 files
'   next      patch narrow model guard

Railmark

A persistent left rail becomes the CSC signature. It is simple, highly usable, and cheap to implement across Ink rows.

Strong fit

  • Excellent scan speed
  • Works in ANSI terminals
  • Low custom component cost

Risk

  • Memory mark is structural, less iconic
B. Cut Seal
 [c/sc]  CSC ready
   /     plan accepted
 01 read    src/utils/model/providers.ts
 02 decide  OpenAI mapping boundary
 03 apply   type-safe provider guard

Cut Seal

A compact CSC seal with a diagonal split. It is more iconic than a rail, suitable for boot, thinking, completion, and permission prompts.

Strong fit

  • Most distinctive
  • Easy to repeat as small mark
  • Can use pure text fallback

Risk

  • Needs restraint to avoid visual noise
C. Pulse Grid
csc  .... thinking

┌ plan ─────────────────────────┐
│ 01 inspect      done          │
│ 02 patch        running       │
│ 03 verify       queued        │
└───────────────────────────────┘

Pulse Grid

A rhythmic dot/grid language for progress and agent activity. It gives CSC a living terminal feel without heavy animation.

Strong fit

  • Good for long-running work
  • Clear progress states
  • Reusable in plans and tools

Risk

  • More component tuning than the others

Initial recommendation

Use Cut Seal as the compact identity mark, then combine it with the Railmark layout rule for actual information display. That gives CSC a memorable symbol while keeping day-to-day terminal output fast to scan and cheap to build.
